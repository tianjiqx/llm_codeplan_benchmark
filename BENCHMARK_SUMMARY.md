### 3.3 关键发现

1. **最快生成速度**: GLM-4.5-Air (Haiku) 在复杂 Prompt 下达到 215 tok/s，表现出色
2. **最佳编码模型**: Qwen3 Coder Next 综合性能优秀，TTFT 低且 TPS 高
3. **火山引擎新模型**: Doubao-Seed-2.0-Code 生成速度高达 98 tok/s，但 TTFT 较高
4. **并发稳定性**: DeepSeek V3.2 在各并发级别下保持 100% 成功率
5. **限流问题**: GLM Code Plan 在并发 10 时出现 50% 限流
6. **⚠️ 新发现 (2026-03-30)**: Qwen3.5 Plus 简单 Prompt TTFT 异常长（119s），需进一步排查
7. **多次运行统计**: 新版本支持 3 次重复测试，提高数据准确性，显示标准差

## 四、异常数据分析

### 4.1 Qwen3.5 Plus 简单 Prompt 异常

| 指标 | 简单 Prompt | 复杂 Prompt | 差异 |
|------|-------------|--------------|------|
| TTFT | 119.56s (±20.98s) | 7.92s (±0.88s) | **15x 差异** |
| 生成 TPS | 5137 tok/s | 61 tok/s | **83x 差异** ⚠️ |
| 端到端 TPS | 53 tok/s | 53 tok/s | 相同 ✅ |

**可能原因**:
1. 模型对简单 prompt 也进入了深度思考模式（类似 DeepSeek thinking）
2. `usage.outputTokens` 统计异常，可能包含了 thinking tokens
3. API 服务端行为变化

**验证建议**: 建议对比单次运行的详细日志，检查 `outputTokens` 数值是否异常。

## 五、建议

### 代码生成场景
- **推荐**: Qwen3 Coder Next (百炼) - 低延迟、高 TPS
- **备选**: GLM-4.5-Air (GLM Code Plan) - 最高生成速度

### 长文本处理
- **推荐**: GLM-4.5-Air 或 GLM-4.7 - 复杂 Prompt 下表现最佳
- **备选**: DeepSeek V3.2 (火山引擎) - 稳定性好（注意 thinking 模式）

### 高并发场景
- **推荐**: DeepSeek V3.2 - 各并发级别稳定
- **注意**: GLM Code Plan 高并发下可能触发限流

### 测试建议
- 使用 `repetitions: 3` 配置提高数据准确性
- 对比 thinking 模型和非 thinking 模型时，注意 TTFT 统计差异
- 简单 Prompt 测试可帮助识别模型深度思考行为
